================================================================================
                              Data Flow Diagram
================================================================================

NORMAL OPERATION:
-----------------
User → ALB → Spot Target Group → Fargate Spot Service (2 tasks)
                              ↓
                    Spot Success Monitor
                              ↓
                    Reset error count (DynamoDB)

FAILOVER FLOW:
--------------
1. Spot Task Failed (STOPPED event)
2. EventBridge → Spot Error Detector Lambda
3. Error count incremented in DynamoDB
4. Threshold reached (≥3 errors)
5. Spot Error Detector → Failback Orchestrator Lambda (async)
6. Failback Orchestrator:
   - Start Standard Fargate Service (0→2 tasks)
   - Wait for stable
   - Stop Spot Service (2→0 tasks)
   - Update failover_state in DynamoDB
   - Send SNS notification
7. ALB automatically routes to Standard Target Group

RECOVERY FLOW:
--------------
1. Spot Task Started (RUNNING event)
2. EventBridge → Spot Success Monitor Lambda
3. Reset error count in DynamoDB
4. Check failover_state (active)
5. Spot Success Monitor → Cleanup Orchestrator Lambda (async)
6. Cleanup Orchestrator:
   - Wait CLEANUP_DELAY (30s)
   - Start Spot Service (0→2 tasks)
   - Wait for stable
   - Stop Standard Service (2→0 tasks)
   - Clear failover_state in DynamoDB
   - Send SNS notification
7. ALB automatically routes back to Spot Target Group

================================================================================
                              Network Flow
================================================================================

Inbound:
  User → Internet → ALB (Public Subnet) → Target Group → ECS Tasks (Private)

Outbound:
  ECS Tasks (Private) → NAT Gateway (Public) → Internet

Note: ECS Tasks have no public IP, all outbound traffic goes through NAT

================================================================================
                           IAM Permissions Flow
================================================================================

Lambda Execution Role:
  ├── ECS Access: UpdateService, DescribeServices
  ├── DynamoDB Access: GetItem, PutItem, UpdateItem (specific table only)
  ├── SNS Access: Publish (specific topic only)
  └── Lambda Invoke: InvokeFunction (specific functions only)

ECS Task Role:
  ├── CloudWatch Logs: CreateLogStream, PutLogEvents
  └── (Minimal permissions based on application needs)
